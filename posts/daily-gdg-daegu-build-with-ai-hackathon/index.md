# [Daily] GDG Daegu Build With AI ハッカソンに参加してみました！

생성일: 2026년 3월 23일 오후 7:55

---

![GDGIcon.png](/images/daily/gdg-daegu-build-with-ai-hackathon/logo.ico)

## 概要

---

大邱で取材するGoogle Developer Groupでのハッカソンを参加しました！

AIでビルドをし、自分たちのサービスを作ろうと言う大会でした！

普段AIを使って開発はしていましたが、短期間でどのぐらい作れるかを試してみたかったので興味を持ちました！

なので自分はここで友達３人と一緒にチームを組んで大会に参加することになりました。

## 内容

---

自分たちはまずアイディアをジムに通ってるユーザーをターゲットにトレーニングの面倒を見てくれるアプリケーションを企画しました。

まずこのアイディアーにした理由は二つです。

1. 初心者がジムで運動をする時、自分の姿勢があってるかわからない。
2. Youtubeで姿勢を見ても真似するのが難しい

この二つの問題を発見し、これを土台として開発を進みました。

主に開発時間は２時間半で、サービスリリーズや、発表の準備までやる感じで進みました。

![IMG_7007.JPG](/images/daily/gdg-daegu-build-with-ai-hackathon/IMG_7007.JPG)

自分はここで社会社であり、開発者として役割を担当しました。

### アイディア確定過程

---

開発時間は 13:00 ~ 15:30までだったので、早めにアイディアを決めたかったです。

なので自分は 13:00 ~ 13:15まで開発に必要な

1. サービスの目的
2. サービスの概要
3. ターゲットユーザー
4. 必要な機能
5. あって欲しい機能

などを決めました。

サービスの目的として、ジムに通ってる人たちのためにガイドラインの提供。

そして姿勢を矯正してくれる！のが自分たちのサービスの目的でした。

このサービスはジムに興味がある人たちが最初運動マシンを使いづらいと言ってくれる時が多く、自分の姿勢があっているかわからない時が多いという周りの言葉でこのようなサービスが必要だと判断しました。

なので自分たちはターゲットユーザーを運動初心者、老人、運動をやっていますが自分の姿勢があっているかわからないと言う人たちをターゲットにしました。

ここで必要な機能として、姿勢を把握できるAI、そして自分たちの姿勢を正しく判断できるのかの研究が必要でした。

姿勢を判断するためにはスキャン. つまり自分たちの姿勢をカメラで取り、それをAIに入れ込んでフィードバックを残せるよう機能を作ることが大事でした。

その後はログイン機能や、自分が残した記録に対してアクセスできるようMypageがあって欲しいという意見を交換しました。

### サービス内容

---

![logo.ico](/images/daily/gdg-daegu-build-with-ai-hackathon/logo.ico)

議論の結果、自分たちのサービスでGymbroと言う名前のサービスをリリーズすることに決まりました。

中で使っていた AI は Gemini AI や Media Pipe を使用しました。

実際にYoutubeで正しい姿勢の画像をとり、それをMedia Pipeに入れて人の骨格の画像を作りました。

これをプロムポートで記入し、この姿勢が正しいのかフィードバックを残して欲しいと言う研究を1時間を使ってしました。

![스크린샷 2026-03-20 135115.png](/images/daily/gdg-daegu-build-with-ai-hackathon/%EC%8A%A4%ED%81%AC%EB%A6%B0%EC%83%B7%202026-03-20%20135115.png)

![[KR] 피지컬 갤러리 Youtube](/images/daily/gdg-daegu-build-with-ai-hackathon/%EC%8A%A4%ED%81%AC%EB%A6%B0%EC%83%B7%202026-03-20%20135156.png)

[KR] 피지컬 갤러리 Youtube

> 初期のプロムポート

```
[System Role]

あなたはウェイトトレーニング初心者のためのAI姿勢矯正コーチです。

ユーザーの目標は、正確なターゲット筋肉（胸）の刺激点を見つけ、誤った姿勢による肩の痛みを予防することです。

[Task]

添付された画像を分析して、ベンチプレス姿勢の問題点を把握してください。 特にMediaPipeが描いた白い骨格オーバーレイを参考にして、左右のバランス、肘の角度、肩甲骨の固定を評価してください。

[Input Context]

現在の画像はベンチプレスの最後のプレス（上げる）区間です。

[Output Format (JSON)]

必ず以下のJSON形式でのみ回答してください。

{

  "posture_score": "0~100の間のスコア",
  "danger_level": "Safe / Warning / Danger 중 택 1",
  "ai_visual_analysis": "画像と骨組みを通じて把握した視覚적特徴 (1文)",
  "primary_issue": "最も核心的な問題点 (1文)",
  "feedback_message": "初心者が即座に姿勢を修正できる具体的な行動指針（2〜3文）"
}
```

> 結果プロムポート

```
{
  "posture_score": "98",
  "danger_level": "Safe",
  "ai_visual_analysis": "白い骨格オーバーレイの分析結果、下降時の肘の角度が適切であり、特に上昇時にも肩関節のノードがベンチに固定され、肩甲骨の後降下設定が完璧に維持されています。"
  "primary_issue": "大きな問題は見つかりませんでした。 初心者が目指すべき理想的な姿勢です。"
  "feedback_message": "提示された正しい例は、ベンチプレスの教科書的な姿勢です。 肩甲骨のしっかりとした固定、適切な肘の角度、そして動作中ずっと維持される胸椎の伸展まで完璧です。 この姿勢を体に覚えさせ、毎セットこの感覚を再現しようと努力することが、胸の筋肉のターゲット刺激を見つけ、肩の痛みを予防する近道です。"
}
```

上のようにMedia Pipeで伝えといた写真の点数や、フィードバックをもらってユーザーに渡して画面に写せるようページを作る計画を立てました。

### ページ構成

---

自分たちは主に下のようなアーキテクチャで作業する感じで進みました。

![Architecture.png](/images/daily/gdg-daegu-build-with-ai-hackathon/Architecture.png)

自分はサーバーのプロジェクト構築や、フロントエンドの作業に入りました。

サーバーの方はバックエンドエンジニアを勉強している2人のチームメートがいたので、残っているフロントエンドをやりました。

１時間内でVercelへのリリーズや、Next.jsと画面スキャンをGemini AI と Antigravityを利用して作りました。

ここでGithub Organizationで開発を進めていましたが、OrganizationはVercel無料バージョンでリリーズができない理由があり、個人RepositoryへコードをコピーできるようGithub Action의 코드를書いて問題を解決しました。

したは自分が作ったページです！

ちなみに１時間だと書きましたが、Notion MCPを繋いでContextを記入すると一瞬でページを作ってくれるのがすごく不思議でした！
